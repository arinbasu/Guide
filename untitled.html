Preamble

You can control a web based writing system and presentation system entirely offline and opportunistic availability of going online using a state of the art version control. This is particularly valuable in public health and epidemiology where we need to work as new data becomes available. Also, for conducting a continuously updated meta analysis, this is a valuable approach. My intention here is to put a step by step guide on using Authorea and Jupyter notebooks to do reproducible data science in public health that brings in not just dynamically updating a document but doing so seamlessly and brining in the power of a team to provide necessary insights throughout the document \cite{Peng_2011}. 
In brief,  the steps are as follows:
Step 1: Start with Authorea and set up an article
Step 2: Link the article to github or a git repo, do a remote add origin to the github to your local hard drive on a folder after installing git locally
Step 3:  Start a jupyter notebook and use the python module (python 2 and python 3). You can use R as well; I'd say if you want to use a notebook and want to use R, use an Rnotebook in Rstudio; the effects will be same for the workflow that I outline here.
Step 6: Use git to push the ipynb into the Authorea repo; ditto for Rnotebook markdown to push tables, figures, and text generated in the Rnotebook.
I have included a demo with this a tutorial on data analysis with jupyter notebook with python 3
Keep in mind these general principles:
Write in markdown and for tables use csv files and for figures use jpg files
Store the csv files in subfolders in the figures folder 
Store the jpeg files in subfolders in the figures folder
Use the layout.md file to position the various
In Authorea itself, it does not matter what format you use to write your text.

Step 1: Start with authorea

The purpose of this paper is to build a dynamically updated paper as new information comes in and new analyses are conducted. We would need some dynamically updated parts and some consistent parts in such a document. We also need to collaborate and share the document, keep everything open so that the source code of the analyses remain in public domain. In any scholarly or information sharing publication, you need four elements:
  • You need to produce tables of information
  • You need to provide your readers visualisation of elements
  • You need to attribute your arguments or statements with sources (citations)
  • For reproducible research you need to provide source codes of your analyses (here we have used python but you can use R or Julia as well).
Mixing Authorea with dynamically developed graphs and tables is a great way to work. You can do all of these in the Authorea main interface by uploading from your local hard drive but you may also want to work on the web on the fly and here github is your friend.
